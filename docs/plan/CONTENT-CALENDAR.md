# GOBIYA — Page Content Plan (no blog)

All content lives on outcome pages, problem pages, and tool pages. No blog, no articles. Depth comes from making each page the definitive answer for its query.

## Problem pages — target queries & angles

| Page | Target queries | Angle |
|---|---|---|
| /problems/invisible-to-ai | "why doesn't chatgpt mention my business", "how to get cited by AI", "brand not in AI overviews" | Diagnose: no entity graph, no citable claims, blocked AI crawlers. Tool: AI Visibility Checker. |
| /problems/traffic-no-leads | "high traffic no conversions", "website traffic but no sales" | Diagnose: cognitive load, intent mismatch, funnel friction. Outcome: traffic-that-converts. |
| /problems/google-penalty | "google manual action recovery", "site traffic dropped after core update" | Diagnose: link profile, thin content, spam history. Tool: Aged Domain Lookup. |
| /problems/pages-not-indexed | "pages discovered not indexed", "index bloat", "programmatic pages not ranking" | Diagnose: crawl budget, quality gates, duplication. Tool: Index Bloat Analyzer. |
| /problems/losing-clicks-to-ai | "AI overviews stealing clicks", "CTR dropped AI overviews" | Diagnose: ranked-but-skipped content. Outcome: cited-by-ai. |

## Outcome pages — evidence requirements

Each outcome page needs 3 proof tiles (`.proof-grid`) with **real numbers** and 1 short client evidence block (industry + metric + timeframe — no fluffy testimonials). Placeholder stats from the homepage (412%, 188%, 97.4) must be replaced with or backed by real engagement data before launch.

## Tool page copy (each 600–1,000 words, below the tool)

1. What the tool checks and why it matters (quotable, specific — this is what LLMs cite)
2. Methodology (data sources, limits — E-E-A-T signal)
3. How to read the results
4. FAQ (3–5) → FAQPage schema

## Voice & style

Match the homepage register everywhere: declarative, systems language, zone/status metaphors (`// PROBLEM 02 — PIPELINE STATE: LEAKING`). Short sentences. Numbers over adjectives. Every page ends at the deploy CTA.
